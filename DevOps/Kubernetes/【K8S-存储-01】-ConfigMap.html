ConfigMap-配置管理 ​

一、ConfigMap介绍 ​

  • ConfigMap用于保存配置数据，以键值对形式存储。
  • ConfigMap资源提供了向Pod注入配置数据的方法。
  • ConfigMap旨在让镜像和配置文件解耦，以便实现镜像的可移植性和可复用性。

二、创建ConfigMap的4种方式 ​

1、使用字面值创建 ​

shell
kubectl -n xx create configmap my-config --from-literal=key1=config1 --from-literal=key2=config2
kubectl -n xx get cm
kubectl -n xx describe cm my-config

Data
===
key1:
---
config1
key2:
---
config2

2、使用文件创建 ​

shell
kubectl -n xx create configmap my-config-2 --from-file=/etc/resolv.conf
key的名称是文件名称，value的值是该文件内容

Data
===
resolv.conf:
---
nameserver 114.114.114.114

3、使用目录创建 ​

shell
mkdir test && cp /etc/resolv.conf test/ && cp /etc/fstab test/
kubectl -n xx create configmap my-config-3 --from-file=test
目录中的文件名为key，文件内容是value
kubectl -n xx describe cm my-config-3

4、使用yaml创建 ​

yaml
cm.yml
apiVersion: v1
kind: ConfigMap
metadata:
  name: cm1-config
  namespace: xx
data:
  db_host: "172.25.0.250"
  db_port: "3306"
  
kubectl apply -f cm.yml
kubectl -n xx describe cm cm1-config

三、如何使用ConfigMap ​

1、使用ConfigMap设置环境变量 ​

yaml
pod1.yml
apiVersion: v1
kind: Pod
metadata:
  name: pod1
  namespace: xx
spec:
  containers:
    - name: pod1
      image: busyboxplus
      command: ["/bin/sh", "-c", "env"]
      env:
        - name: key1
          valueFrom:
            configMapKeyRef:
              name: cm1-config    #值来源于上文中建立的cm1-config中的db_host
              key: db_host
         - name: key2
           valueFrom:
             configMapKeyRef:
               name: cm1-config
               key: db_port
  restartPolicy: Never
  
  kubectl apply -f pod1.yml
  kubectl -n xx logs pod1
  key1=172.25.0.250
  key2=3306

2、直接导入变量-不改变key ​

yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod2
  namespace: xx
spec:
  containers:
    - name: pod2
      image: busyboxplus
      command: ["/bin/sh", "-c", "env"]
      envFrom:
        - configMapRef:
            name: cm1-config
  restartPolicy: Never
  
  kubectl apply -f pod2.yml
  kubectl -n xx logs pod2
  db_host=172.25.0.250
  db_port=3306

3、使用ConfigMap设置命令行参数 ​

yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod2
spec:
  containers:
    - name: pod2
      image: busyboxplus
      command: ["/bin/sh", "-c","echo $(db_host) $(db_port)"]
      envFrom:
        - configMapRef:
            name: cm1-config
  restartPolicy: Never
  
  kubectl -n xx logs pod2
  172.25.0.250 3306

4、通过数据卷使用ConfigMap ​

yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod3
spec:
  containers:
    - name: pod3
      image: busyboxplus
      command: ["/bin/sh", "-c", "ls -l /config/"]
      volumeMounts:  
      - name: config-volume  #挂此卷
        mountPath: /config   #在容器内自动创建路径
  volumes:
    - name: config-volume  #当挂此卷的时候映射到cm1-config
      configMap:
        name: cm1-config
  restartPolicy: Never
  
  kubectl -n xx logs pod3

四、ConfigMap热更新 ​

shell
kubectl -n xx edit cm cm1-config
kubectl -n xx describe cm cm1-config

Data
====
db_host:
----
172.25.0.100
db_port
----
8080

修改卷对应的cm值，Pod数据并不会实时更新，需要等待几秒

五、ConfigMap热更新后需手动触发Pod滚动更新 ​

yaml
default.conf
server {
    listen       8080; 
    server_name  _;

    location / {
        root /usr/share/nginx/html;
        index  index.html index.htm;
    }
}
kubectl create configmap nginx-config --from-file=default.conf

demo.yml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: demo
spec:
  replicas: 1
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myapp:v1
        volumeMounts:
        - name: config-volume
          mountPath: /etc/nginx/conf.d
      volumes:
        - name: config-volume
          configMap:
            name: nginx-config
  kubectl apply -f demo.yml && curl 10.244.141.210:8080
  kubectl edit cm nginx-config (80)
  edit修改后，删除pod，重建生效。
